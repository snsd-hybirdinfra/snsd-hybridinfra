# Repository Orientation

This document gives a short map of the repository for reviewers and contributors.

## Repository model

The repository has three main layers:

- scenarios for lifecycle-aligned validation cases
- labs for implementation boundaries and runtime validation
- reports for quality, coverage, and evidence summaries

## Recommended reading order

1. Start with [README.md](../README.md).
2. Read this orientation document for the overall map.
3. Open [labs/README.md](../labs/README.md) and [scenarios/README.md](../scenarios/README.md).
4. Use [validation-reports/README.md](../validation-reports/README.md) for review summaries.

## Key directories

- [labs/](../labs/) — implementation-oriented lab boundaries and runtime scripts
- [scenarios/](../scenarios/) — lifecycle-aligned operational scenarios
- [docs/](./) — stable reviewer documentation
- [validation-reports/](../validation-reports/) — concise review summaries
- [reports/](../reports/) — detailed validation diagnostics
