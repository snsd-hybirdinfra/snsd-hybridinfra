# Semantic Section Taxonomy

## Purpose

Semantic sections define operational visualization zones used by Diagram Renderer V2.

A section is not a visual decoration.

A section represents a semantic operational responsibility area.

---

# Core Principle

Renderer V2 must render operational meaning before visual aesthetics.

Sections exist to improve:

- reviewer readability
- operational reasoning visibility
- lifecycle comprehension
- workflow interpretation
- governance understanding

---

# Shared Rendering Rules

## Rules

- every node belongs to a semantic section
- section boundaries must remain visually visible
- section order must preserve lifecycle readability
- operational flow direction must remain explicit
- governance sections should remain visually distinct
