# Ansible Automation Lab Evidence

This directory stores lab-local evidence generated during validation execution.

## Evidence Focus

Ansible inventory, SSH access, playbook execution, rollback, recovery, validation, and automation evidence.

## Evidence Areas

| Directory | Purpose |
|---|---|
| raw/ | Raw Ansible command output, playbook logs, SSH checks, and execution records |
| processed/ | Parsed playbook results, normalized validation outputs, and recovery state summaries |
| summary/ | Reviewer-readable automation evidence summaries |

## Current Status

| Area | Status |
|---|---|
| Documentation Status | documentation-ready |
| Implementation Status | planned |
| Execution Status | not yet executed |
| Evidence Status | placeholder until lab execution |

## Evidence Policy

Evidence files in this directory should be generated from actual Ansible execution once the lab is implemented.

Before implementation, this directory defines the expected evidence boundary only.
